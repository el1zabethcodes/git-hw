# Reflection

**1. Навіщо Git у сучасних проєктах?**  
Git потрібен для командної розробки, щоб зберігати історію змін, уникати втрати коду та мати змогу працювати над різними частинами проєкту одночасно. Він допомагає координувати роботу кількох людей і впорядковує процес розробки.

**2. Мій досвід після виконання завдання**  
Під час роботи я навчилась створювати SSH-ключ, налаштовувати віддалений репозиторій і виконувати повний цикл роботи з гілками та комітами. Тепер я розумію, як саме Git і GitHub взаємодіють між собою.

**3. Що було найзручнішим / найважчим**  
Найзручніше — працювати через GitHub-інтерфейс із Pull Request, бо там усе наочно.  
Найважчим спочатку було розібратись із SSH-налаштуванням, але тепер я впевнено працюю з ключами.

**4. Як це застосувати у курсовому або колективному проєкті**  
У курсовому проєкті я планую створити окремі гілки для різних модулів програми, робити Pull Request для кожної зміни та вести чітку історію комітів. Це допоможе підтримувати порядок у коді та ефективно співпрацювати з іншими.
